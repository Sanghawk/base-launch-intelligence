# Stage 9 — Expansion

**Goal:** Add leverage only after the core loop works.

**What needs to be accomplished:**

- Expand only from validated user demand.
- Improve wallet-cohort models if users trust and use the core ranking system.
- Add deeper Farcaster-native workflows if social linkage proves valuable.
- Add team dashboards, saved views, APIs, or paid alerts if users ask for them.
- Add historical research mode if users need retrospective analysis.
- Consider other chains only after the Base-specific thesis is proven.
- Avoid expanding into features that dilute the core launch-intelligence workflow.

**Gate:** Are you expanding because users demand it, or because the roadmap feels more exciting than the core product?
