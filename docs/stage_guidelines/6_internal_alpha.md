# Stage 6 — Internal Alpha

**Goal:** Use the product manually and find where it breaks.

**What needs to be accomplished:**

- Run the system against live Base launches.
- Track false positives, missed launches, bad scores, stale data, provider failures, and noisy alerts.
- Manually inspect whether the score explanations make sense.
- Compare your ranked view against your normal manual workflow.
- Fix canonicalization, scoring, and alerting problems before adding more features.
- Determine whether the product saves time or creates more analysis burden.

**Gate:** Does it save analyst time or produce better-ranked observability?
