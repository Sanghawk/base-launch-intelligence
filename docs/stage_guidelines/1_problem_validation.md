# Stage 1 — Product Discovery

**Goal:** Validate that the problem is real before building.

**What needs to be accomplished:**

- Identify the first user type and their workflow.
- Understand how users currently discover, filter, and monitor new Base token launches.
- Compare the current workflow against DEX Screener, GeckoTerminal, Basescan, Farcaster, and manual research.
- Determine whether the pain is discovery speed, scam avoidance, liquidity assessment, deployer history, or signal overload.
- Validate whether users would actually use or pay for a better-ranked monitoring workflow.

**Gate:** Do users actually need this, or are you just building a cool data product?
