# Stage 2 — Product Definition

**Goal:** Convert the idea into a concrete product spec.

**What needs to be accomplished:**

* Define the first MVP scope.
* Specify the core user workflow from token discovery to score interpretation to alert action.
* Define the minimum feature set needed to evaluate new Base launches.
* Decide which metrics matter most: contract risk, deployer reputation, liquidity quality, holder concentration, wallet-cohort overlap, and Farcaster-linked attention.
* Define success metrics and kill criteria before implementation starts.
* Cut anything that does not directly support launch triage and quality ranking.

**Gate:** Is the MVP narrow enough to build and test quickly?
